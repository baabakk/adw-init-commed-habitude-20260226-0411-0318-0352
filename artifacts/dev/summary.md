This workspace implements a TypeScript habit-tracking platform with reminder delivery, analytics, auth, and multi-channel messaging (SMS, Telegram, WhatsApp), including both API/server logic and a Vite-based frontend. The stack is Node.js + Express-style routing, TypeScript (compiled to `dist/` with `.d.ts` outputs), a SQL-backed data layer (`schema.sql`, `src/database.ts`, `server/database.ts`), and a React frontend in `frontend/*.tsx` with static/public pages in `public/`. Core backend domains are split across `src/routes/*` (habits, reminders, completions, analytics, users, auth), `src/services/*` (scheduler, messaging gateway/providers, reminder delivery, streaks), and model/data modules (`src/models/*`, `src/db/*`, `src/data-store.ts`), with a parallel `server/*` implementation for auth/habits/webhooks services. Notable patterns include adapter/provider abstractions for channel-specific messaging (`src/adapters/*`, `src/services/*-provider.ts`), scheduled reminder orchestration (`src/services/reminder-scheduler.ts`, `server/services/scheduler.ts`), shared middleware for auth/error handling, and duplicated build outputs under both `dist/` and `dist/src/` indicating multiple compile targets or layered packaging.
